## Project

**Name:** CRSH-NXS (crsh-nexus)
**Description:** Central registry / federation directory for city crash-risk
engines — forks of the PHLCRSH-V2 mapping template. Lets visitors browse
registered city instances, view a federation map, and register a new fork via
a form that writes to localStorage (permanent listing requires a PR to this
repo).
**Target Platform:** web (static site, GitHub Pages)
**Primary Language:** JavaScript (JSX)
**Framework:** React 19 + Vite 8

component directory, no state library. Everything lives in one component.

## Key Architectural Patterns

- **Single-component app:** `App.jsx` owns all state via `useState`/`useMemo`
  — node registry, theme, search/filter, modal/form state. No context, no
  reducer, no external state library. Don't split this into multiple
  components/contexts unless the file becomes genuinely unwieldy — it's
  intentionally simple for a small static directory site.
- **Client-only "registration":** submitting the registration form only
  appends to `nodes` state and persists to `localStorage`
  (`crsh-nexus-nodes`). There is no backend — permanent listing requires a
  human to open a Pull Request adding a config object to `initialNodes` in
  `src/App.jsx`. Don't build a backend for this without being asked.
- **Design language (road-infrastructure theme):** established in this
  session — Barlow Condensed (display) / Overpass (body) / Overpass Mono
  (data), sign-yellow accent (`--yellow`), dashed "centerline" motif
  (`--centerline` gradient) reused across header, footer, modal head, map
  connections, and the deployment-step connector. All tokens are CSS custom
  properties in `src/index.css` — extend the palette there, don't hardcode
  colors in JSX.
- **SVG federation map:** hand-rolled inline SVG in `App.jsx` (not a mapping
  library) — node positions are manually set `x`/`y` coordinates (0–1000,
  0–500 viewBox) entered via the registration form.

## Common Gotchas

1. **Vite `base: '/CRSH-NXS/'`** — the app is served from a subpath on GitHub
   Pages. Local dev preview must be visited at `/CRSH-NXS/`, not `/`, or you'll
   see a blank page / 404s for assets.
2. **Dev server port** — `vite.config.js` reads `PORT` env var, falling back
   to 5180. If 5180 is already in use by another local process, Vite will
   auto-increment (`strictPort: false`).
3. **Registering an engine doesn't persist globally** — it only writes to the
   current browser's `localStorage`. Don't mistake a successful form
   submission for an actual repo/data change; the user must open a PR to make
   it permanent.
4. **Pushing to `main` deploys to production** — the GitHub Actions workflow
   builds and force-pushes `dist/` to `gh-pages` on every push to `main`.
   There is no staging environment.

## How to Navigate by Task

| If you need to... | Go to... |
|--------------------|------------------|
| Change any UI, layout, or copy | `src/App.jsx` (everything lives here) |
| Change colors, type, spacing, or the road-motif styling | `src/index.css` |
| Add/modify a registered city engine's default data | `initialNodes` array in `src/App.jsx` |
| Change the activity/milestone log | `initialFeed` array in `src/App.jsx` |
| Change the registration form fields | `nexus-form` section in `src/App.jsx` + matching `nexus-form-*` classes in `src/index.css` |
| Change dev port or GH Pages base path | `vite.config.js` |
| Change the deploy pipeline | `.github/workflows/*.yml` |
